# 셰이더 플레이그라운드 WBS

## 1. 문서 목적

이 문서는 `shader_playground_development_plan.md`를 기반으로 셰이더 플레이그라운드 개발을 실제 작업 단위로 분해한 WBS(Work Breakdown Structure) 문서이다.

목표는 다음과 같다.

- 구현 범위를 실제 개발 가능한 단위로 쪼갠다.
- 선행 작업과 후속 작업의 관계를 명확히 한다.
- 각 작업의 완료 기준을 정의한다.
- 스프린트 계획, 일정 산정, 진행률 관리의 기반으로 사용한다.

이 문서는 일정표 자체가 아니라 **작업 분해 문서**이다. 일정은 이 문서의 작업 ID를 기준으로 별도 관리한다.

---

## 2. WBS 작성 기준

- 작업 ID는 계층 구조를 표현한다.
- 너무 큰 작업은 하위 작업으로 계속 분해한다.
- 각 작업은 가능한 한 “완료 여부를 판단할 수 있는 단위”로 작성한다.
- 각 작업에는 설명, 선행 작업, 산출물, 완료 기준을 포함한다.
- 문서/테스트/리팩터링도 구현 작업과 동일한 중요도로 관리한다.

---

## 3. 최상위 WBS 개요

- 1. 프로젝트 기반 구축
- 2. 렌더링 코어 구축
- 3. 셰이더 컴파일 및 실행 시스템
- 4. 코드 에디터 UI
- 5. 머티리얼 프로퍼티 인스펙터
- 6. 텍스처 업로드 및 샘플러 연결
- 7. 기본 프리뷰 씬 및 카메라
- 8. 모델 업로드 및 모델 프리뷰
- 9. 저장/불러오기 및 프로젝트 상태 관리
- 10. 에셋 관리 UI
- 11. 에러 처리 및 디버깅 도구
- 12. 성능 최적화 및 안정화
- 13. 문서화 및 예제 자산
- 14. WebGPU 확장 대비 구조 정리

## 6. MVP 완료 정의

다음 조건을 만족하면 1차 MVP 완료로 본다.

- 사용자가 vertex/fragment shader 코드를 편집할 수 있다.
- 셰이더를 컴파일하고 에러를 확인할 수 있다.
- active uniform 기반으로 프로퍼티 UI가 자동 생성된다.
- 사용자가 이미지를 업로드해 sampler2D에 연결할 수 있다.
- fullscreen quad 또는 기본 도형에 셰이더를 적용할 수 있다.
- fbx 업로드 후 모델 프리뷰가 가능하다.
- 현재 작업 상태를 저장하고 다시 복원할 수 있다.

## 7. 후속 활용 방법

이 WBS 문서는 다음 작업의 기반으로 사용한다.

- 스프린트 계획 수립
- 칸반/이슈 트래커 작업 분할
- 개발 일정 추정
- 진행률 체크리스트 생성
- 테스트 범위 정의

권장 방식은 각 WBS 항목을 이슈 트래커의 Epic / Task / Subtask 단위로 매핑하는 것이다.
